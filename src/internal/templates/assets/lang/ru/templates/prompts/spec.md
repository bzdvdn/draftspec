# Prompt спецификации Draftspec

Вы создаете или обновляете одну feature specification.

## Goal

Сформируйте ясную спецификацию фичи в `.draftspec/specs/<slug>.md`, которая соответствует конституции.

## Load Only

- `.draftspec/constitution.md`
- `.draftspec/memory.md`
- текущий запрос пользователя и разговор
- минимальный контекст репозитория, необходимый для снятия неоднозначности

## Do Not Read By Default

- нерелевантные спецификации
- нерелевантные plan packages
- тяжелые implementation-части кода, если они не нужны для корректного определения scope
- contracts или data models других фич

## Stop Conditions

Остановитесь и задайте минимальный уточняющий вопрос, если:

- цель фичи неоднозначна
- критерии приемки пришлось бы выдумывать, а не выводить из запроса
- запрошенная фича выглядит конфликтующей с конституцией

Не переходите к планированию или реализации, пока сама спецификация не прояснена.

## Rules

- Спецификация ДОЛЖНА соответствовать конституции.
- Держите спецификацию сфокусированной на одной фиче или одном изменении.
- Используйте настроенный в проекте язык документации для нового или обновляемого содержимого спецификации.
- Уважайте уже сложившееся локальное соглашение документа только если без этого сохранение существующего файла станет непоследовательным.
- Не смешивайте языки заголовков и секций внутри одной спецификации без сильной причины.
- Используйте конкретные критерии приемки и явные границы scope.
- Задавайте уточняющие вопросы только когда недостающая информация действительно критична.
- Не переходите здесь к детальному implementation planning.

## Output expectations

- Запишите или patch-обновите `.draftspec/specs/<slug>.md`
- Кратко суммируйте цель, scope, критерии приемки и открытые вопросы
