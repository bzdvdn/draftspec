# Prompt обзора проекта Draftspec

Вы формируете обзор проекта на уровне всего workspace для новой агентской сессии.

## Goal

Дать краткое актуальное резюме проекта и всех активных фич, чтобы новая сессия могла сориентироваться без повторного чтения каждого артефакта.

Эта команда опциональна, не требует slug и не создаёт файлов — только inline-ответ в разговоре.

## Phase Contract

Inputs: `.draftspec/constitution.md`; spec-файлы и inspect-отчёты для активных фич.
Outputs: только inline-ответ — файл не записывается.
Stop if: `.draftspec/constitution.md` отсутствует.

## Load First

Всегда сначала читайте:

- `.draftspec/constitution.md`

## Load If Present

Читайте только для определения фазы и статуса каждой фичи:

- `.draftspec/scripts/list-specs.sh` — запустите для перечисления активных спеков; исходник не читайте
- Только заголовочный блок каждого активного спека (цель и маркеры фазы — не читайте спек целиком)
- Только metadata block из `.draftspec/specs/<slug>/inspect.md` (строка verdict), если inspect существует

## Do Not Read By Default

- Полное содержимое спеков, кроме заголовка и индикаторов фазы
- Plan packages, если пользователь явно не просит детали уровня плана
- tasks.md, data-model.md, contracts/
- Архивированные фичи
- Implementation-файлы
- Исходники скриптов

## Stop Conditions

Остановитесь и задайте вопрос только если:

- `.draftspec/constitution.md` отсутствует

## Правила

- Прочитайте конституцию один раз; за один проход извлеките назначение проекта, ограничения и ключевые принципы.
- Запустите `list-specs` для получения списка активных спеков. Если скрипт недоступен, перечислите файлы в `.draftspec/specs/`.
- Для каждого активного спека читайте только то, что нужно, чтобы извлечь: slug, название фичи, текущую фазу и вердикт inspect, если он доступен.
- Определяйте текущую фазу по набору присутствующих артефактов: нет inspect → фаза spec; inspect есть, нет plan → фаза inspect; plan есть, задачи открыты → фаза plan, tasks или implement; все задачи закрыты → фаза verify или archive.
- Не читайте полный plan package ни для одной фичи, если пользователь явно не просит.
- Держите ответ компактным: он должен читаться меньше чем за 30 секунд.
- Если активных фич нет, явно скажите об этом после краткого описания проекта.

## Формат ответа

Отвечайте только inline. Используйте такую структуру:

**Проект** — одно или два предложения из конституции: что это за проект и его основное ограничение или цель.

**Активные фичи** — одна строка на фичу:

```
<slug>  <name>  [фаза]  [inspect: pass|concerns|blocked — опустить, если inspect нет]
```

**Итого** — одна строка: сколько фич активно, сколько заблокировано, какая фаза преобладает.

**Рекомендуемая следующая команда** — одно наиболее полезное следующее действие исходя из текущего состояния (например, команда восстановления для наиболее заблокированной фичи или команда для фичи, ближайшей к архивированию).

Держите весь ответ в пределах 20 строк, если в проекте не более 5 активных фич.

## Self-Check

- Читал ли я только минимально нужное для определения фазы каждой фичи?
- Достаточно ли компактен ответ для быстрой ориентации новой сессии?
- Избегал ли я чтения полного содержимого спеков или планов?
- Основана ли рекомендуемая следующая команда на реальном состоянии фич?
