# Prompt архивации Draftspec

Вы архивируете один feature package.

## Goal

Создайте устойчивый архивный снимок одной фичи, или восстановите ранее архивированную фичу обратно в активную разработку.

## Flags

`--copy`: оригиналы остаются на месте после архивации (режим только копирования). Полезно для `deferred` фич, которые могут вернуться в активную разработку.

`--restore`: обратная операция — копирует последний снимок обратно в активные `specs/` и `plans/`, затем удаляет запись из архива. См. Restore Rules ниже.

## Phase Contract

Inputs: смотрите Load First и Load Only If Needed.
Outputs: архивный снимок и summary (режим по умолчанию); восстановленные активные файлы (режим restore).
Stop if: смотрите Stop Conditions.

## Load First

**Режим по умолчанию** — всегда сначала прочитайте:

- `.draftspec/specs/<slug>/spec.md`

**Режим restore** (`--restore`) — читайте архивный снимок:

- `.draftspec/archive/<slug>/` — используйте самую свежую датированную директорию

## Load If Present

Читайте plan artifacts только чтобы сформировать `summary.md`. Если summary достаточно написать по spec — не читайте их:

- `.draftspec/plans/<slug>/plan.md`
- `.draftspec/plans/<slug>/tasks.md`
- `.draftspec/plans/<slug>/data-model.md`
- `.draftspec/plans/<slug>/contracts/`
- `.draftspec/plans/<slug>/research.md`

## Do Not Read By Default

- нерелевантные archive entries
- нерелевантные файлы репозитория

## Stop Conditions

Остановитесь и задайте минимальный уточняющий вопрос, если:

- неясен статус архивации (режим по умолчанию)
- отсутствует причина архивации (режим по умолчанию)
- целевой slug неоднозначен
- `--restore` используется, но архив для slug не существует
- `--restore` используется, но активные файлы для slug уже существуют (перезаписали бы активную работу)

## Rules

- Архивируйте в `.draftspec/archive/<slug>/<YYYY-MM-DD>/`.
- Поведение по умолчанию — **move-based**: копировать артефакты в директорию архива, затем **удалить** активные файлы (`specs/<slug>/spec.md` и `plans/<slug>/`). Это поддерживает активный workspace чистым.
- Если пользователь передаёт `--copy`, оригиналы остаются на месте (режим только копирования).
- Записывайте `summary.md` внутри директории архива.
- Держите `summary.md` компактным. Предпочитайте статус, причину, завершенный scope и заметные отклонения вместо длинного ретроспективного повествования.
- Если plan artifacts существуют, архивируйте их вместе со spec.
- Если `research.md` не существует, не выдумывайте его.
- Если status равен `completed` и `tasks.md` существует, используйте `/.draftspec/scripts/verify-task-state.* <slug>` перед архивацией. Не заявляйте completed-архив, если обязательные задачи еще открыты.
- Используйте один из этих статусов:
  - `completed`
  - `superseded`
  - `abandoned`
  - `rejected`
  - `deferred`

## Restore Rules

Когда `--restore` присутствует в аргументах пользователя:

- Найдите самый свежий снимок в `.draftspec/archive/<slug>/` (по имени датированной директории).
- Если активные файлы для slug уже существуют (`specs/<slug>/spec.md` или `plans/<slug>/`), остановитесь и спросите пользователя — восстановление перезаписало бы активную работу.
- Скопируйте `spec.md` обратно в `.draftspec/specs/<slug>/spec.md`.
- Скопируйте plan artifacts (`plan.md`, `tasks.md`, `data-model.md`, `contracts/`, `research.md`) обратно в `.draftspec/plans/<slug>/`, если они присутствуют в снимке.
- Не копируйте `summary.md` или `inspect.md` обратно — восстановленная фича должна быть переинспектирована.
- После успешного восстановления удалите запись из архива (датированную директорию). Если это был единственный снимок, удалите также директорию slug из `archive/`.
- Сообщите, какие файлы были восстановлены, и предложите следующую workflow команду на основе восстановленного состояния (обычно `/draftspec.inspect <slug>`).

## Output expectations

### Режим по умолчанию

- Создайте архивный снимок
- Запишите или patch-обновите `summary.md`
- Кратко суммируйте архивированные файлы, статус и причину
- Явно указывайте, что archive — это терминальный шаг workflow для этой фичи, если только пользователь не просит дальнейшую работу

### Режим restore

- Перечислите все восстановленные файлы и их назначения
- Укажите, какой архивный снимок был использован и подтвердите его удаление
- Предложите следующую workflow команду (обычно `/draftspec.inspect <slug>`)
