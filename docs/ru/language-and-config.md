# Языки и Конфигурация

## Поддерживаемые языки

Сейчас Draftspec поддерживает:

- `en`
- `ru`

## Языковые настройки

Основной config хранит четыре языковых значения:

```yaml
language:
  default: en
  docs: en
  agent: en
  comments: en
```

### `default`

Базовый язык, который используется, когда более точные настройки не переопределены.

### `docs`

Управляет генерируемой проектной документацией, например:

- `constitution.md`
- `memory.md`
- шаблонами spec
- шаблонами plan
- шаблонами tasks

### `agent`

Управляет генерируемыми prompts и guidance, которое вставляется в `AGENTS.md`.

### `comments`

Фиксирует предпочитаемый язык новых или измененных комментариев в коде во время `implement`.

## Канонические BDD-маркеры

Даже когда `docs` установлен в `ru`, Draftspec оставляет `Given / When / Then` каноническими acceptance markers. Остальной текст вокруг них должен следовать выбранному языку документации.

## Файл конфигурации

Основной config лежит в `.draftspec/draftspec.yaml` и хранит:

- layout путей
- языковые настройки
- включенные agent targets
- имена templates
- имена scripts

Если config и workspace начинают расходиться, используй `draftspec doctor`.
