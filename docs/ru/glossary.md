# Glossary

## Constitution

Главный документ проекта. Он задает неоспоримые правила для архитектуры, workflow, языковой политики и governance.

## Spec

Документ уровня фичи, который описывает, что должно быть построено и зачем.

## Inspect

Фаза проверки, которая смотрит spec и связанные артефакты на полноту, согласованность и соответствие конституции.

## Plan

Техническая фаза проектирования, которая переводит одну spec в артефакты, пригодные для реализации.

## Plan Package

Набор файлов под `.draftspec/plans/<slug>/`, обычно включающий `plan.md`, `tasks.md`, `data-model.md`, `contracts/` и optional `research.md`.

## Tasks

Исполнимая декомпозиция feature plan. В Draftspec файл `tasks.md` живет внутри plan package.

## Implement

Фаза исполнения, которая стартует от незавершенных задач и обновляет память проекта и состояние tasks.

## Archive

Фаза и область хранения, которые используются для сохранения исторического снимка завершенного или неактивного feature package.

## Memory

Текущая рабочая память проекта, которая хранится в `.draftspec/memory.md`. Она суммирует active state, важные решения и статусы архивированных фич.

## Data Model

Plan-артефакт, который описывает сущности, структуры, связи и важные invariants, нужные для реализации.

## Contracts

Plan-артефакты, определяющие интерфейсы, например API, события или другие внешние границы взаимодействия.

## Research

Опциональный plan-артефакт, который нужен только тогда, когда нужно сохранить неопределенность, внешнее исследование или архитектурный tradeoff.

## Agent Target

Поддерживаемая агентная экосистема, для которой Draftspec может генерировать project-local command или prompt files, например `claude`, `codex`, `copilot`, `cursor`, `kilocode` или `trae`.

## Orphaned Agent Artifact

Сгенерированный agent file, который все еще лежит на диске, хотя его target уже не включен в `.draftspec/draftspec.yaml`.

## Docs Language

Настроенный язык, который используется для генерируемой проектной документации, такой как `constitution.md`, `memory.md`, specs, plans и tasks.

## Agent Language

Настроенный язык, который используется для generated prompts и guidance в `AGENTS.md`.

## Comments Language

Настроенный предпочитаемый язык для новых или измененных комментариев в коде во время реализации.

## Given / When / Then

Канонические BDD-маркеры, которые используются в acceptance criteria независимо от языка документации. Текст вокруг них при этом может следовать настроенному docs language.
